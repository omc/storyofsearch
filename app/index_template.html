Chapter 1

Human vs. Book

There's a lot of text in the world
And we need to find things.
Specific topics, phrases, and even individual words
Searching for these can be a time consuming process.
Page by page you search, hoping to find what you're looking for.
This could go on for a long time.
But for many books, there's a tool to make the process more efficient...

The Index!

It's a lot of work up front...
...but now yo can find what you're looking for alphabetically...
...and then jump to the pages that is references.
This same problem exists in computing!

And we've solved it with the exact same concept. let's jump into what that looks like.

Chapter 2: Computer vs. Database

There's a lot of data in the world
And databases are huge
We need to find a record ina database table.
The simple solution is slow...
Row by row, the program scans hte text for the term we're looking for...
This could go on for a long time.
But we have a concept to make this process efficient...

The Index!

Rows on a table are transformed into an index.
Like a book's index, terms are listed alphabetically (sorted).
A reference to the original database row is attached to the term
Now, when the program searches for a term, it heads over to the index.
Using binary search (more later), it quickly jumps to the term in the list.
Using a scoring system based on the data gathered in the idnexing process, the search yields results sorted by relevancy.
If needed, the program can use the databases reference to grab the original document, or row.
