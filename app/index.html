The Story of Search

Chapter 1

Humans & Books


There's a lot of text in the world.

And we need to find things.

Specific topics, phrases, and even individual words.

Searching for these can be a time-consuming process.

Page by page you search, hoping to find what you're looking for.

This could go on for a long time.

But for many books, there's a tool to make the process more efficient:

The Index!

Creating an index takes a lot of work up front. Someone (or something) goes through the trouble of creating an alphabetized list of words with their page location.

But now you can find the topic you are looking for —

and jump to the pages that the index references.

The same problem exists in computing, and we've solved it with the same concept.

Let's jump in and talk about what this looks like!

Chapter 2

Computers & Databases

There's a lot of data in the world

And databases store a lot of text.

If we need to find something,

the solution is slow.

Row by row, the computer scans the text for the terms we're looking for.

This could go on for a long time.

But we have a concept to make this process efficient.

You guessed it—

The Index!

Row by row, the text in the database is transformed.

This process is called indexing.

Like in the back of a book, the index is organized for fast lookup.

For every term that's inserted into the index, a reference to the original row in the database is attached for later use.

Now, when the computer searches for a term, it skips the database and heads straight to the exact match that's present in the index!

This results in faster and better search results.

Let's jump into a real example with some data!

Chapter 3

Create an Index

During indexing, the search engine analyzes the text from the database, transforming the raw text into a set of "tokens". The tokens are added to the index with meta-data about each record. The index may store how frequently the token is found in a record, and even positions of tokens in the source text. The index also tracks the total number of documents which have a given token, which is used for ranking search results. This structure allows for fast and relevant search results for a given token.
